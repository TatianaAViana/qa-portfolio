# 🧪 QA Portfolio – Manual Testing Project

## 📌 Project Objective

This repository showcases my QA portfolio, containing practical examples of manual testing activities and test documentation.

The goal is to demonstrate how I approach software quality, from planning to execution and reporting.

---

## 📂 Contents

* Test Program
* Test Plan
* Test Procedures
* Test Cases (BDD - Given/When/Then)
* Incident Reports (Bug Reports)
* Test Status Reports
* Test Completion Reports
* Logs and Execution Evidence

---

## 🧠 Testing Approach

This project includes:

* Exploratory testing based on hypotheses
* Validation of user inputs and system behavior
* Analysis of user feedback and error handling
* Identification and documentation of defects
* Focus on usability and responsiveness

---

## 🧪 Example Scenarios Covered

* Invalid form submissions (missing or incorrect data)
* Product creation and deletion flows
* User feedback (error/success messages)
* Authentication and access control
* Responsive layout issues
* Browser-specific behavior (e.g., Firefox issues)
